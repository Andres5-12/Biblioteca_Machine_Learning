Data Engineering Overview

The current pipeline implements the data engineering steps needed to produce a clean, integrated dataset from heterogeneous public library sources.

Ingestion

Data is extracted from the Colombian open data portal using APIs with pagination to handle large datasets safely.

  • Four sources are ingested individually.
  • Requests are performed with limit and offset controls.
  • Each raw dataset is captured before transformation.

Cleaning

Each dataset is cleaned with source-specific rules, including column normalization and standardization of categorical values.

  • Column names are converted to snake_case and stripped of accents.
  • Department and municipality names are normalized and capitalized consistently.
  • Numeric fields such as collection volume and registered users are coerced to numeric type.

Integration

After cleaning, the datasets are concatenated into a master table with a unified schema.

  • Missing columns are added with null values to preserve consistent structure.
  • Duplicate records are removed based on library name, municipality, and department.
  • Source metadata is preserved for traceability.

Enrichment and Export

The master dataset is enriched with derived attributes and exported to CSV/Excel for reporting or future modeling.

  • Region labels are derived from department names.
  • Connectivity flags are normalized across datasets.
  • Final outputs include both raw tables and summary sheets for analysis.
