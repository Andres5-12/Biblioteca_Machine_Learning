Model Development Overview

This page documents the full model development process applied to the Colombian public library dataset. The goal is to train models that predict two important library attributes: whether a library has internet connectivity (classification) and the expected number of registered users (regression).

Target Variables and Problem Definition

The modeling effort focuses on two prediction tasks grounded in the public library dataset:

  • Classification Task: Predict whether a library has internet connectivity based on location, library type, and operational characteristics. Target: has_internet (binary).
  • Regression Task: Predict the number of registered users a library will have based on department, region, collection volume, and service type. Target: usuarios_registrados (continuous).

Dataset Source and Features

The data comes from four Colombian public library networks:

  • Red Nacional de Bibliotecas Públicas: National directory of public libraries.
  • BiblioRed Bogotá D.C.: Public library network in Bogotá.
  • Red Departamental de Bolívar: Department of Bolívar's public library network.
  • Bibliotecas Públicas de Casanare: Department of Casanare's public libraries.

Key features extracted from the dataset include department, municipality, library type, natural region, collection volume, and internet availability flags.

Dataset Split

The integrated library dataset is split into training and validation sets to support model training and unbiased evaluation.

  • Typical split: 80% training, 20% validation on the ~1000+ library records.
  • Ensure random state is fixed for reproducible train/validation folds.
  • For classification, stratify by internet_connectivity to preserve class balance.
  • Handle missing values in user registrations using mean imputation or forward-filling techniques.

Feature Engineering for Libraries

Features are prepared from library metadata to create robust inputs for model training:

from sklearn.preprocessing import StandardScaler, LabelEncoder

# Encode categorical library attributes
encoder = LabelEncoder()
X['departamento_encoded'] = encoder.fit_transform(X['departamento'])
X['region_encoded'] = encoder.fit_transform(X['region_natural'])

# Normalize numeric features
scaler = StandardScaler()
X['volumen_coleccion_scaled'] = scaler.fit_transform(X[['volumen_coleccion']])

# Create derived feature: connectivity by region
connectivity_by_region = df.groupby('region_natural')['tiene_internet_flag'].mean()
X['region_connectivity_rate'] = X['region_natural'].map(connectivity_by_region)

Training Process

Models are trained on library features to predict connectivity and user registration targets:

from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor

# Classification: Predict internet connectivity
clf = RandomForestClassifier(n_estimators=100, max_depth=8, random_state=42)
clf.fit(X_train, y_train_connectivity)
pred_connectivity = clf.predict(X_val)

# Regression: Predict registered users
reg = RandomForestRegressor(n_estimators=100, max_depth=10, random_state=42)
reg.fit(X_train, y_train_users)
pred_users = reg.predict(X_val)

Hyperparameter Configuration

Each model is optimized for the library classification and regression tasks:

  • Classification (Internet Connectivity): Maximize recall for underrepresented libraries; use class_weight='balanced' to handle imbalanced connectivity data.
  • Regression (User Count): Minimize MAE to avoid large prediction errors for library planning purposes.
  • Tree Depth: Limit depth to 8-10 to avoid overfitting on the relatively small library dataset (~1000 records).
  • Feature Importance: Track which library attributes (region, department, collection volume) most strongly influence predictions.

Validation Process

Validation is performed on the library test set using domain-relevant metrics:

  • Classification Validation: Accuracy, precision, recall for connectivity predictions; important for resource allocation decisions.
  • Regression Validation: MAE and R² for user count predictions; helps identify which libraries will have high engagement.
  • Cross-validation with 5 folds ensures stability across different library samples.
  • Residual analysis reveals which regions or library types are harder to predict accurately.

Model Summaries

Each candidate model is evaluated for its ability to support library planning and policy decisions:

  • Logistic Regression: Interpretable predictions for connectivity; useful for explaining to library administrators why a facility is predicted to have/not have internet.
  • Random Forest: Captures complex patterns in regional library characteristics; identifies which attributes matter most for connectivity and user growth.
  • Gradient Boosting: Strong predictive power for user count estimation; helps allocate resources to libraries with high predicted engagement.
