Model Engineering Overview

Model engineering is the phase where candidate algorithms are selected, features are prepared, and validation strategies are defined.

Candidate Models

For a typical public library analytics problem, both classification and regression models are considered. The project evaluates three strong models for each task:

  • Classification: Logistic Regression, Random Forest Classifier, Gradient Boosting Classifier.
  • Regression: Linear Regression, Random Forest Regressor, Gradient Boosting Regressor.

Feature Engineering

The feature engineering process focuses on creating robust inputs for modeling by standardizing categorical values and deriving meaningful attributes.

  • Normalize department and municipality names for consistent categorical encoding.
  • Convert numeric fields to proper numeric type and fill missing values in a pipeline-safe way.
  • Create derived features such as region group, connectivity availability, and library activity indicators.

Validation Strategy

A reliable model selection process uses cross-validation and holdout test sets to compare candidate models with the same data splits.

  • Use a holdout validation set or cross-validation to ensure evaluation metrics are stable.
  • Compare models using classification and regression metrics appropriate to the problem type.
  • Choose the model that balances accuracy, robustness, and interpretability for production implementation.
